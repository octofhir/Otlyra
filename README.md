# Otlyra

An experimental web browser engine, written from scratch in Rust.

Currently: a native window on macOS with a GPU-presented frame rasterized through
the engine's own painting seam. No HTML, no CSS, no DOM, no text yet.

## Requirements

macOS on Apple Silicon. Rust 1.96.0, pinned by `rust-toolchain.toml`.

## Running

```sh
cargo run                                   # open the window
cargo run -- --screenshot frame.png         # render one frame and exit
just --list                                 # every other task
```

`--screenshot` needs no display server and exits non-zero on failure, so it is
what drives the image tests in CI.

## Crates

| Crate | Owns |
|---|---|
| `otlyra-app` | The shell: entry points, tracing, the placeholder scene |
| `otlyra-platform` | Window, GPU surface, event loop. The only crate that knows what an OS is |
| `otlyra-gfx` | The `PaintTarget` seam and its rasterizer backends |

Two rules hold the layering together, and both are enforced by tests rather than
by convention:

- No `winit` or `wgpu` type appears in `otlyra-platform`'s public API, so a
  windowing or GPU major version bump stays inside that crate.
- `PaintTarget` has two backends from the first commit — Skia and a recorder — so
  the choice of rasterizer stays reversible.

## Development

```sh
just ci        # fmt, clippy, test, deny, audit, screenshot
just metrics   # cold build time, binary size, package count
```

Supply-chain gates need `just install-tools` once.
