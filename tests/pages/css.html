This page is the CSS counterpart to defaults.html. It states what each rule above should do, so that the difference between "matched" and "applied" stays visible while the cascade is being built.

What works today

Selector matching, through Servo's own engine over our DOM: type, class, id, attribute (including ^=, $=, *=), the four combinators, structural pseudo-classes, and :is(), :where() and :not(). The --dump-selectors flag answers "which elements does this selector match" against a real document.

What does not work yet

The cascade: nothing in the <style> block above changes this page, and neither would a style= attribute. Specificity, inheritance, !important, calc(), custom properties and media queries all arrive with it.
