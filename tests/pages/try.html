Everything to press

One page for trying things by hand. Each block says what should happen.

Suggestions

Press the field: the list drops. Type ber: it narrows, without case and anywhere in the word. Arrows walk it and mark where you are without filling the field; return takes what is marked, escape leaves it. Pressing a suggestion takes it.

Sliders

Press anywhere on a track: the thumb goes there. Drag it, wander off the track — it still follows. Arrows step, page keys ten steps, home and end reach the ends. The last one steps by five, so it lands on 0, 5, 10 and nothing between.

Bars

Nothing to press. The second one has no value: it says a task is going on without saying how far, so it stays empty rather than reading as zero. The meters differ only in where their optimum sits, which is what decides the colour.

Dates and times

Press one: it takes hold of a part. Type digits — 7 in a month is July and moves on, because no month starts with a seven. Left and right walk the parts, up and down step the one in hand and wrap at its ends, backspace empties it. A part missing, or the thirtieth of February, means the control holds nothing — send the form and see.

Colour

It shows what it holds. The third says nonsense, and nonsense is black.

Files, and a form that carries one

Press one: the machine's own dialogue opens. Dismiss it and the control keeps what it held. The second takes several and says how many; the third asks for text files. Then send the form — it is a POST spelled multipart/form-data, so the file goes with it. Start something to send it to first (see below); the answer that comes back is whatever that prints.

Upload

The same answers in an address

A GET, so everything above turns into a query and the address bar shows it. A slider with nothing typed into it still sends the middle of its range; a half-filled date sends nothing; a file sends its name and not its contents.
